name: Commit and push at meaningful changesets
description: User wants commits and pushes only at approved meaningful milestones, not incrementally
type: feedback
---

Commit and push once we have an approved meaningful changeset — not after every small change.

**Why:** User prefers clean, purposeful commit history over granular incremental commits.
**How to apply:** Batch related changes, get user approval, then commit and push together. Don't commit intermediate states.
