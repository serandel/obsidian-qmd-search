description: Never use real note/file names from the user's vault in public-facing content like issues or PRs
type: feedback
---

Do not use real note names or file paths from the user's vault when creating public content (GitHub issues, PRs, etc.). Use made-up examples with foo/bar/baz instead.

**Why:** Privacy — the user doesn't want their personal note titles exposed publicly.

**How to apply:** When writing issues, PRs, or any public content that references vault files, always substitute with generic placeholder names.
