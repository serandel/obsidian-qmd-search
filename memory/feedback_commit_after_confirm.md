---
name: Commit after user confirms
description: Always commit changes after the user confirms they work, without needing to be asked
type: feedback
---

After the user confirms a change is working, commit it immediately without being asked.

**Why:** The user expects each confirmed change to be committed right away. They had to remind me twice.

**How to apply:** When the user says a change looks good / works / is confirmed, create the commit. Don't wait for an explicit "commit this" instruction.
